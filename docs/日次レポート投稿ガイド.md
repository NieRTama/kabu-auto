# 日次レポート投稿ガイド（X / Discord）

kabu-auto は平日17:30に当日/週次/月次/総合の損益サマリ・勝率を **X（旧Twitter）** と
**Discord** へ自動投稿できる（どちらも任意機能・未設定なら何もしない。両方同時に有効化も可）。
本書は両プラットフォームの認証情報の取得手順・設定方法・投稿内容・制約をまとめたもの。

> **どちらを使うべきか**: Discordは完全無料でWebhook URLのみで動作し、APIキー取得や
> 審査が不要。Xは2026年時点で無料枠が無く完全従量課金（投稿1件あたり約$0.015。URLを
> 含む投稿は約$0.20）だが、1日1回・URL無しの投稿なら月額1ドル未満（約50円程度）。
> **無料で済ませたいだけならDiscordのみで十分**。Xにも投稿したい場合のみXの設定を追加する。

---

## 1. 何のための機能か

`alert()`（異常時アラート）とは別に、日次の振り返り用として当日・週次・月次・総合の
損益（円・基準資金に対する%）と勝率を1日1回投稿する。`x.enabled` / `discord_report.enabled`
それぞれ独立しており、有効化したものだけが投稿される。

## 2. Discordの設定（無料・推奨）

### 2.1 Webhook URLの取得

1. 投稿先にしたいDiscordサーバーのテキストチャンネルを開く。
2. チャンネル設定（歯車アイコン）→「連携サービス」→「ウェブフック」→「新しいウェブフック」。
3. 任意の名前を設定し、「ウェブフックURLをコピー」を押す。

> 通知（異常時アラート）用の `DISCORD_WEBHOOK_URL` とは**別のURLを使うことを推奨**
> （日次レポートと異常アラートが混ざらないよう、別チャンネルに分けられる）。
> 同じURLを設定すれば同一チャンネルにまとめることもできる。

### 2.2 設定方法

`.env` に追記:

```
DISCORD_REPORT_WEBHOOK_URL=<コピーしたWebhook URL>
```

`config.yaml` で機能を有効化:

```yaml
discord_report:
  enabled: true
```

## 3. Xの設定（従量課金）

### 3.1 X API認証情報の取得方法

1. [X Developer Portal](https://developer.x.com/) にログインし、開発者アカウントを作成する（既存のXアカウントが必要）。
2. 新しいプロジェクト・アプリを作成する。
3. アプリの「User authentication settings」で **OAuth 1.0a** を有効化し、
   App permissions を **「Read and Write」** に設定する（既定は読み取り専用のため、
   必ず変更すること。投稿には書き込み権限が必須）。
4. 「Keys and tokens」タブで以下の4つを発行・コピーする:
   - API Key（Consumer Key）
   - API Key Secret（Consumer Secret）
   - Access Token
   - Access Token Secret

> **重要**: 後から App permissions を「Read」→「Read and Write」に変更した場合、
> 既存のAccess Token/Secretは古い権限のままなので、**変更後に必ずAccess Tokenを
> 再生成**してコピーし直すこと（再生成しないと投稿時に権限エラーになる）。

### 3.2 設定方法

`.env` に追記:

```
X_API_KEY=<API Key>
X_API_SECRET=<API Key Secret>
X_ACCESS_TOKEN=<Access Token>
X_ACCESS_TOKEN_SECRET=<Access Token Secret>
```

`config.yaml` で機能を有効化:

```yaml
x:
  enabled: true
```

どちらも設定を反映するには kabu-auto の再起動が必要（起動時に一度だけ読み込む）。

## 4. 損益%の基準資金（モード別・X/Discord共通）

- **paper モード**: `trading.paper_initial_capital`（既定50万円）を自動的に基準にする。設定不要。
- **live / dry_run / semi_live**: 実運用は入出金を自動追跡できないため、ダッシュボードの
  「📣 日次レポート自動投稿（X / Discord）」カードからモードごとに基準資金を手動設定する
  （`reference_capital.json` に永続化される）。**未設定（0円）の間は%が省略され、円額のみ
  表示される。**

## 5. 投稿内容（テンプレート。X/Discord共通）

```
【kabu-auto 日次レポート】
モード: ペーパー（仮想取引）

当日: +12,345円 (+2.47%) 勝率58%
週次: -3,200円 (-0.64%) 勝率45%
月次: +45,600円 (+9.12%) 勝率52%
総合: +123,456円 (+24.69%) 勝率55%
```

- 損益0円・取引0件などで勝率が算出できない期間は「勝率」表示を省略する。
- 基準資金が未設定の期間は `(...%)` を省略し、円額のみ表示する。
- 文字数上限を超える場合は安全に末尾を切り詰める（投稿自体が失敗するより、要点が
  欠けても投稿が成功する方を優先）。**X=280文字・Discord=2000文字**で上限が異なる
  （Discordの方が余裕があるため通常は切り詰め不要）。

ダッシュボードの「🐦 X投稿テキストを確認」「💬 Discord投稿テキストを確認」ボタンで、
実際に送信せずに今の集計結果でのテキストをいつでも確認できる。

## 6. 実装の挙動・制約

- **失敗は飲み込む**: 投稿APIが失敗（認証エラー・レート制限・ネットワーク障害等）しても
  アプリは継続し、`logger.error` にのみ記録する。再送・リトライは行わない。
- **認証情報はログに出さない**: 投稿テキストはログに出るが、APIキー・Webhook URL等は出力しない。
- **投稿は1日1回**: 平日17:30のみ（X・Discordとも同時刻）。リアルタイム性は求めず、日次の振り返り用途。
- **集計対象**: `DRY_RUN`ステータスの取引（実発注していない記録）は損益集計から除外される。
- **X/Discordは完全に独立**: 片方だけ有効化、両方有効化、いずれも可能。一方の送信失敗が
  他方に影響しない。

## 7. 動作確認方法

1. ダッシュボードの確認ボタンで内容を確認する（`enabled`の値に関わらず確認可能。
   実際の投稿は行われない）。
2. 実際に投稿されるか確認したい場合は、Python REPLから直接呼び出せる:

```python
import src.core.config as cfg
cfg.load("config.yaml")

# Discord（無料）
from src.core import discord_report
discord_report.post_daily_report("paper", 500_000)

# X（従量課金）
from src.core import x_poster
x_poster.post_daily_report("paper", 500_000)
```

投稿が届かない場合は、まずログに `X投稿失敗` / `Discord日次レポート投稿失敗` が
出ていないか確認する（認証情報・URLの誤り、権限不足、レート制限等が主な原因）。
