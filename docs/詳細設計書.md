### 1.7 src/strategy/ml_model.py — 機械学習モデル

**責務**: LightGBMモデルの学習・保存・推論

#### `train(df)` — 学習フロー

```
1. build_features() で特徴量・ラベル生成
2. データが100件未満 → ValueError
3. TimeSeriesSplit(n_splits=5) でCV
   - 各foldでLGBMClassifier学習
   - early_stopping(20) で最適ステップ数を取得
4. CV精度をログ出力
5. 最適ステップ数で全データを使って最終モデル学習
6. models/lgb_model.pkl に保存
```

**LightGBMパラメータ**:

| パラメータ | 値 |
|-----------|-----|
| n_estimators | 200（early stoppingで調整） |
| learning_rate | 0.05 |
| num_leaves | 31 |
| random_state | 42 |

#### `predict_proba(model, df)` — 推論

```
入力: OHLCV DataFrame
  → build_features() で特徴量生成
  → 最新行のみ抽出
  → model.predict_proba() で上昇確率取得
出力: float（0.0〜1.0）
  ※ データ不足時は 0.5（中立）を返す
```

### 1.8 src/strategy/signal.py — シグナル生成

**責務**: ルールベース＋MLスコアを統合して売買シグナルを生成

#### ルールベーススコア計算

| 条件 | スコア変動 |
|------|----------|
| ゴールデンクロス（ma5がma25を上抜け） | +0.4 |
| デッドクロス（ma5がma25を下抜け） | -0.4 |
| RSI < 30（売られすぎ） | +0.3 |
| RSI > 70（買われすぎ） | -0.3 |
| 終値 < ボリンジャー下限 | +0.2 |
| 終値 > ボリンジャー上限 | -0.2 |
| MACDヒストグラム正転換 | +0.1 |
| MACDヒストグラム負転換 | -0.1 |

スコア範囲: -1.0 〜 +1.0（`max(-1.0, min(1.0, score))` でクリップ）

#### 統合スコア計算

```
MLスコア変換: (上昇確率 - 0.5) × 2   ← -1.0〜+1.0 に正規化

統合スコア = ルールスコア × 0.5 + MLスコア × 0.5

統合スコア >= +0.6 → BUY
統合スコア <= -0.6 → SELL
それ以外           → HOLD
```

### 1.9 src/risk/manager.py — リスク管理

**責務**: 発注前のバリデーションと損切り判定

**クラス: `RiskManager`**

| メソッド | 説明 |
|---------|------|
| `can_place_order()` | 日次注文数上限チェック |
| `reset_daily_counters()` | 日次カウンタリセット（毎朝呼び出し） |
| `calc_position_size(symbol, price, cash)` | 購入可能株数計算 |
| `check_max_positions()` | 最大保有銘柄数チェック |
| `check_sector_concentration(sector)` | セクター集中率チェック |
| `should_stop_loss(symbol, price)` | 損切りライン判定 |
| `validate_buy(symbol, price, cash, sector)` | 買い注文の総合バリデーション |

#### 購入株数計算

```
上限金額 = 現金残高 × max_position_ratio（デフォルト20%）
最大単元数 = floor(上限金額 / (株価 × 100))
購入株数 = 最大単元数 × 100
```

#### 損切り判定

```
損益率 = (現在値 - 平均取得単価) / 平均取得単価
損益率 <= stop_loss_pct（デフォルト-5%）→ 損切り発動
```

### 1.10 src/execution/order_manager.py — 発注管理

**責務**: 指値注文の発注・ポジション更新・タイムアウト管理

**クラス: `OrderManager`**

#### ペーパーモードとliveモードの切り替え

```python
self._is_paper = config["trading"]["mode"] == "paper"
```

| モード | buy()/sell() の動作 |
|--------|-------------------|
| paper | DBにPENDINGで記録 → 即FILLEDとして処理 |
| live | kabuステーションAPIで発注 → WebSocketで約定確認 |

#### 注文IDの生成

| モード | 形式 |
|--------|------|
| paper | `PAPER-BUY-{symbol}-{uuid8桁}` |
| live | kabuステーションAPIが返す OrderId |

#### タイムアウトキャンセル

```
発注と同時に threading.Timer を設定
  → order_timeout_seconds（デフォルト300秒）後に cancel_order() を呼び出し
  → 約定確認イベントを受信した場合はタイマーをキャンセル
```

#### ポジション更新ロジック（BUY時）

```
既存ポジションあり:
  新平均取得単価 = (既存単価×既存株数 + 新単価×新株数) / (既存株数+新株数)
  株数 += 新株数

既存ポジションなし:
  新規Positionレコードを作成
```

#### ポジション更新ロジック（SELL時）

```
損益 = (売却価格 - 平均取得単価) × 売却株数
株数 -= 売却株数（最小0）
損益を対応するTradeレコードに記録
```

## 2. シーケンス図

### 2.1 通常の売買フロー（スイングトレード）

```
15:35（平日）
    │
    ▼
signal_scan() 実行
    │
    ├─ load_ohlcv(symbol) ── DB
    │
    ├─ gen_signal(symbol, df, model)
    │     ├─ compute_indicators()
    │     ├─ _rule_score()
    │     └─ ml_model.predict_proba()
    │
    ├─ シグナルをDBに保存
    │
    └─ BUYシグナルの場合:
          │
          ├─ risk.validate_buy()
          │     ├─ can_place_order()
          │     ├─ check_max_positions()
          │     └─ check_sector_concentration()
          │
          ├─ risk.calc_position_size()
          │
          └─ order_mgr.buy(symbol, price, quantity)
                │
                ├─ [paper] DBに記録 → ポジション更新
                └─ [live]  kabuAPIで発注 → タイマーセット
                              │
                              └─ WebSocket OrderEvent受信
                                    → タイマーキャンセル
                                    → status = FILLED
```

### 2.2 損切りフロー

```
9:00〜15:30（5分毎）
    │
    ▼
stop_loss_check() 実行
    │
    ├─ client.get_board(symbol) で現在値取得
    │
    └─ risk.should_stop_loss(symbol, price)
          損益率 <= -5%?
                YES → order_mgr.sell()
                     └─ alert("損切り実行", ...)
```

## 3. エラー処理方針

| 状況 | 対応 |
|------|------|
| kabuステーション接続失敗（起動時） | ログ警告を出してペーパーモードで継続 |
| APIトークン更新失敗 | LINE Notifyでアラート送信 |
| WebSocket切断 | 自動再接続（指数バックオフ、最大60秒） |
| 注文発注失敗 | ログエラーを記録、処理スキップ |
| データ取得失敗（yfinance） | ログ警告、当該銘柄をスキップ |
| ML推論失敗 | ログ警告、MLスコアを0.0（中立）として処理を継続 |
| DB操作エラー | SQLAlchemyの例外をcatch、ログ記録 |
