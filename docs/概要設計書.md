## 1. システム概要

### 1.1 目的

日本株（東京証券取引所）を対象としたスイングトレード（数日〜数週間保有）の完全自動売買システム。auカブコム証券のkabuステーションAPIを使用し、ルールベース＋機械学習ハイブリッド戦略でシグナルを生成・発注する。

### 1.2 対象ユーザー

個人投資家（システム管理者兼利用者）

### 1.3 運用形態

| 項目 | 内容 |
|------|------|
| 稼働形態 | 24時間365日常駐起動 |
| 取引時間 | 東証前場 9:00〜11:30 / 後場 12:30〜15:30（平日のみ） |
| 取引モード | paper（ペーパートレード）/ live（本番） |
| 運用資金 | 50万円以下（小規模） |

## 3. 機能一覧

### 3.1 主要機能

| No. | 機能 | 説明 |
|-----|------|------|
| F01 | 市場データ取得 | yfinanceで権利修正済み日足OHLCVを取得・DB保存 |
| F02 | テクニカル指標計算 | MA・RSI・ボリンジャーバンド・MACDを計算 |
| F03 | MLモデル学習 | LightGBMで5日後上昇を予測するモデルを学習 |
| F04 | シグナル生成 | ルールベース＋MLスコアを統合して売買判断 |
| F05 | 発注実行 | kabuステーションAPIで指値注文を発注 |
| F06 | ポジション管理 | 保有銘柄・平均取得単価・損益をDB管理 |
| F07 | リスク管理 | 損切り・最大銘柄数・セクター集中・注文上限を制御 |
| F08 | Webダッシュボード | ポジション・損益・シグナル履歴をブラウザで確認 |
| F09 | 緊急全決済 | ダッシュボードから全ポジションを即時決済 |
| F10 | アラート通知 | 損切り実行・API異常時にLINE Notifyで通知 |
| F11 | 自動バックアップ | SQLite DBを毎日バックアップ（30日分保持） |
| F12 | 週次再学習 | 毎週日曜2:00にMLモデルを自動再学習 |

### 3.2 スケジュール機能

| 時刻 | 処理 | 条件 |
|------|------|------|
| 毎朝 8:30 | APIトークン更新 | 毎日 |
| 9:00〜11:30 / 12:30〜15:30 (5分毎) | 損切りチェック | 平日のみ |
| 15:35 | シグナルスキャン | 平日のみ |
| 16:00 | 市場データ更新 | 毎日 |
| 17:00 | DBバックアップ | 毎日 |
| 毎週日曜 2:00 | MLモデル再学習 | 週次 |

## 6. 取引戦略概要

### 6.1 ハイブリッドシグナル

```
ルールベーススコア（重み 50%）
  + MLスコア（重み 50%）
  = 統合スコア（-1.0 〜 +1.0）

統合スコア >= 0.6  → BUY シグナル
統合スコア <= -0.6 → SELL シグナル
それ以外           → HOLD
```

### 6.2 使用テクニカル指標

| 指標 | パラメータ | 用途 |
|------|-----------|------|
| 移動平均線 | 短期5 / 中期25 / 長期75 | MAクロスでトレンド判定 |
| RSI | 14日 | 売られすぎ(30以下)/買われすぎ(70以上) |
| ボリンジャーバンド | 20日 / 2σ | バンド外れで反転シグナル |
| MACD | 標準設定 | ヒストグラム転換でシグナル |

### 6.3 機械学習モデル

| 項目 | 内容 |
|------|------|
| アルゴリズム | LightGBM（二値分類） |
| 予測対象 | 5日後に+2%以上上昇するか（1/0） |
| 特徴量 | テクニカル指標10種 |
| 学習方式 | 時系列交差検証（5分割）→全データで最終学習 |
| 再学習 | 毎週日曜2:00 |

## 7. リスク管理概要

| リスク | 対策 |
|--------|------|
| 過大投資 | 1銘柄あたり最大20%（設定変更可） |
| 急落 | 損切りライン-5%で強制売却 |
| 過集中 | 同一セクター40%上限 |
| 流動性不足 | 指値注文＋5分タイムアウトキャンセル |
| 過剰発注 | 1日の注文数上限100件 |
| ギャップ | 最大銘柄数5銘柄に制限してリスク分散 |
| DB損失 | 日次バックアップ（30日分） |
| システム異常 | ダッシュボードから緊急全決済ボタン |

## 8. 非機能要件

| 項目 | 要件 |
|------|------|
| 可用性 | 24/365稼働（PC起動中） |
| パフォーマンス | シグナル生成 < 1秒/銘柄 |
| セキュリティ | パスワードは環境変数または設定ファイルで管理 |
| 拡張性 | ウォッチリスト銘柄はconfig.yamlで変更可能 |
| 保守性 | loguru によるローテーションログ（30日分保持） |

## 9. 制約・前提条件

- kabuステーション（Windowsアプリ）が常時起動していること
- PCのスリープを無効化していること（スリープでAPI切断）
- kabuステーションのAPIトークンは毎朝8:30に自動更新
- WebSocketは切断時に自動再接続（最大60秒間隔）
- 土曜1〜5時はkabuシステムメンテナンスのため処理を停止
