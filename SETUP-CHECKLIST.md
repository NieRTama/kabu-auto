# セットアップ・チェックリスト（購入者向け）

本ソフトウェアを安全に使い始めるための確認事項です。順に確認してください。

## 利用の前提（必須環境）
- [ ] **Windows PC**（kabuステーションはWindows専用）
- [ ] **auカブコム証券の口座**を開設済み
- [ ] **kabuステーション**をインストールし、API利用を申込・有効化済み
- [ ] Python 3.11 以上をインストール済み（ソース実行の場合）

## 初期設定
- [ ] `.env` に `KABU_API_PASSWORD` を設定した（`.env.example` を参照）
- [ ] `config.yaml` の `trading.mode` が **`paper`（ペーパー）** になっていることを確認した
- [ ] ウォッチリスト（`watchlists.json` またはGUI）に対象銘柄を登録した
- [ ] 過去データを取得し、MLモデルを学習させた（README「初期データ・モデル学習」参照）

## セキュリティ
- [ ] ダッシュボードは既定で `127.0.0.1`（ローカルのみ）。LAN内の他端末から使う場合のみ `dashboard.host` を `0.0.0.0` にする
- [ ] LAN公開すると**ログイン認証**が有効になることを理解した
- [ ] LAN公開後の初回アクセスで、ログイン画面の「初期設定」からユーザーID・パスワードを作成した（`data/auth.json` にPBKDF2ハッシュで保存される）
- [ ] パスワードは8文字以上に設定し、忘れないよう管理した（復元不可。再設定は `data/auth.json` を削除して再作成）
- [ ] curl等のプログラムから操作する場合は、起動時コンソールの `X-API-Token` を控えた
- [ ] LAN通信は平文HTTPであることを理解し、**信頼できるLAN内でのみ**利用する
- [ ] `data/` 配下（DB・ログ・バックアップ・`auth.json`）へのアクセス権限を必要な人だけに制限した

## ペーパーで十分に検証してから本番へ
- [ ] **数週間以上**ペーパーモードで動作を確認し、想定どおりの売買・損益になることを確認した
- [ ] バックテスト/MLの過去成績は将来を保証しないことを理解した
- [ ] 窓開けにより損切りラインを大幅に超えて約定する場合があることを理解した

## 本番（ライブ）移行
- [ ] `config.yaml` の `trading.mode` を `live` に変更した
- [ ] 環境変数 `CONFIRM_LIVE_TRADING=true` を設定して起動することを理解した
- [ ] ダッシュボードに表示される**ライブモード警告**を確認し、実弾取引であることを理解した
