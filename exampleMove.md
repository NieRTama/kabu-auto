# 動作例：50万円投入時のライブ（本番）取引の挙動

このドキュメントは、ウォッチリストに登録した銘柄について「どういう条件で何株買うか」「どういう条件で売るか」「資金配分はどうなるか」を、実際の資金（例：50万円）を証券口座に入金してライブモードで運用する場合を例に説明する。

> 前提：`config.yaml` の `trading.paper_initial_capital`（[config.yaml](config.yaml)）は**ペーパーモード専用**の仮想資金額であり、ライブモードには影響しない。ライブモードでは毎回 kabuステーションAPI経由で実際の証券口座の余力（`StockAccountWallet`）を取得して発注計算に使うため、証券口座に50万円を入金していれば、その50万円（厳密には日々変動する余力）がそのまま使われる。

## リスクプロファイル

`risk_profiles`（[config.yaml](config.yaml)）で「ローリスク・ローリターン」「ハイリスク・ハイリターン」の2種類を定義しており、ダッシュボードの「⚖️ リスクプロファイル」パネルからいつでも切り替え可能（再起動不要、次回の signal_scan / stop_loss_check から反映）。アクティブなプロファイルは `risk_profile.json` に永続化される。

| 項目 | ローリスク (low_risk) | ハイリスク (high_risk) |
|---|---|---|
| 買い閾値 (combined_score ≥) | 0.14 | 0.08 |
| 売り閾値 (combined_score ≤) | −0.14 | −0.08 |
| 1銘柄あたり最大投資比率 | 余力の12% | 余力の35% |
| 損切りライン | −4% | −10% |
| 最大同時保有銘柄数 | 6銘柄 | 4銘柄 |
| セクター集中上限 | 30% | 55% |
| 当日損失上限 | 15,000円 | 60,000円 |

## 1. 買い条件

対象はウォッチリストの**アクティブなリスト**に登録された銘柄のみ（[src/core/watchlist.py](src/core/watchlist.py)）。

毎日 **16:20**（データ更新16:00完了後、[src/core/scheduler.py](src/core/scheduler.py)）に各銘柄のスコアを計算する。

### ルールベーススコア（rule_score、−1〜+1）

| 指標 | 条件 | 加点 |
|---|---|---|
| MAクロス | 5日線が25日線を上抜け（ゴールデンクロス） | +0.4 |
| MAクロス | 5日線が25日線を下抜け（デッドクロス） | −0.4 |
| RSI | 30未満（売られすぎ） | +0.3 |
| RSI | 70超（買われすぎ） | −0.3 |
| ボリンジャーバンド | 終値が下限バンド未満 | +0.2 |
| ボリンジャーバンド | 終値が上限バンド超 | −0.2 |
| MACD | ヒストグラムがゼロを上抜け | +0.1 |
| MACD | ヒストグラムがゼロを下抜け | −0.1 |

（[src/strategy/signal.py](src/strategy/signal.py) の `compute_rule_score`）

### MLスコア（ml_score、−1〜+1）

LightGBMモデルの予測確率を `(proba - 0.5) * 2` で −1〜+1 に変換（モデル未学習の場合は0）。

### 合成スコア

```
combined_score = rule_score × rule_weight + ml_score × ml_weight
```

既定では `rule_weight = ml_weight = 0.5`（[config.yaml](config.yaml) の `strategy` セクションで調整可能）。

**`combined_score` がアクティブなプロファイルの `buy_threshold` 以上ならBUYシグナル。**

### 発注タイミング

- **ペーパーモード**：signal_scan内でその場（当日終値）で即時シミュレート購入。
- **ライブモード**：signal_scanではシグナルをDBに保存するだけ。**翌営業日 9:05** の `morning_execution`（[main.py](main.py)）で、直近20時間以内に生成されたシグナルに基づき成行（現在値）で発注する。

### 発注前の追加チェック（`validate_buy`、[src/risk/manager.py](src/risk/manager.py)）

いずれか1つでも引っかかると見送り（ログに理由が出力される）：

1. 当日注文数が上限（`daily_order_limit` = 100件/日）未満
2. 当日実現損失が上限未満（プロファイルの `max_daily_loss`）
3. 保有銘柄数が `max_positions` 未満
4. 同一セクターの保有比率が `max_sector_ratio` 未満
5. 計算した購入株数が1単元（100株）以上ある

## 2. 何株買うか（ポジションサイズ）

```
購入株数 = floor( 余力 × max_position_ratio / (現在値 × 100) ) × 100株
```

（[src/risk/manager.py](src/risk/manager.py) の `calc_position_size`。東証の単元株100株単位で切り捨て）

### 例：余力50万円

| プロファイル | max_position_ratio | 現在値2,000円の銘柄 | 現在値500円の銘柄 |
|---|---|---|---|
| ローリスク (12%) | 60,000円 | 0株（1単元20万円必要） | 100株 |
| ハイリスク (35%) | 175,000円 | 0株（1単元20万円必要） | 300株 |

ライブ時の余力は注文ごとに毎回APIで取得し直すため、複数銘柄に買いが入るたびに残り余力は減っていく。

## 3. 売り条件（2種類が独立して動く）

### ① シグナルベースの売り

`combined_score ≤ sell_threshold` でSELLシグナル。保有していれば**保有数量を全株売却**。タイミングは買いと同じ（ペーパーは即時、ライブは翌朝9:05）。

### ② 損切り（ストップロス）

平日 **9:00〜15:30 の間5分おき**に独立してチェック（[src/core/scheduler.py](src/core/scheduler.py)）。保有銘柄の現在値が平均取得単価（avg_cost）から `stop_loss_pct` 以上下落していたら、シグナルと無関係に即座に成行で全株売却（[main.py](main.py) の `stop_loss_check`）。

- ペーパーモード：日足終値で判定
- ライブモード：板の現在値で判定

## まとめ：ハイリスク・ハイリターン設定の場合（余力50万円の例）

| 項目 | 値 |
|---|---|
| 買い閾値 | combined_score ≥ 0.08 |
| 売り閾値 | combined_score ≤ −0.08 |
| 1銘柄あたり最大投資額 | 175,000円（余力の35%） |
| 損切りライン | −10% |
| 最大同時保有銘柄数 | 4銘柄 |
| セクター集中上限 | 55% |
| 当日損失上限 | 60,000円（到達したら以降の新規注文を停止） |
